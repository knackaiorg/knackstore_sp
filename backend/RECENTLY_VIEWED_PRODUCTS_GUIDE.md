# Recently Viewed Products - Backend Implementation Guide

## Overview
This document describes the backend implementation for tracking and displaying recently viewed products for guest users. The feature allows guests to see their browsing history during their session.

## Architecture

### Components Created

#### 1. **Entity Layer** (`ProductView.java`)
Represents a product view record in the database.

**Fields:**
- `id` - Primary key (auto-generated)
- `guestSessionId` - UUID generated by the frontend to identify the guest session
- `product` - ManyToOne relationship to Product entity
- `viewedAt` - Timestamp of when the product was viewed
- `viewCount` - Counter for how many times this product was viewed in this session

**Database Table:** `product_views`

---

#### 2. **Repository Layer** (`ProductViewRepository.java`)
Handles all database operations for ProductView entities.

**Key Methods:**
```java
// Find all views for a session (paginated, ordered by most recent)
List<ProductView> findByGuestSessionIdOrderByViewedAtDesc(String guestSessionId, Pageable pageable)

// Find specific product view for a session
Optional<ProductView> findByGuestSessionIdAndProductId(String guestSessionId, Long productId)

// Clear all views for a session
void deleteByGuestSessionId(String guestSessionId)

// Count total views for a session
long countByGuestSessionId(String guestSessionId)
```

#### 4. **Service Layer** (`ProductViewService.java`)
Implements business logic for product view tracking.

**Key Methods:**

**`logProductView(guestSessionId, productId)`**
- Records a product view for a guest session
- If product was already viewed:
  - Updates `viewedAt` timestamp to current time (moves to front)
  - Increments `viewCount`
  - No duplicate in the list
- If product not yet viewed:
  - Creates new ProductView record
- Automatically cleans up old views beyond the 10-item limit
- Returns LogProductViewResponse

**`getRecentlyViewed(guestSessionId)`**
- Returns up to 10 most recently viewed products
- Ordered by most recent first (largest timestamp first)
- Lightweight response (just essential product info)
- Returns RecentlyViewedResponse

**`getRecentlyViewedWithDetails(guestSessionId)`**
- Returns up to 10 most recently viewed products with full ProductDTO details
- Includes variants, ratings, category information
- Useful when you need comprehensive product data
- Returns List<ProductViewDTO>

**`clearRecentlyViewed(guestSessionId)`**
- Deletes all product view records for a session
- Used when user explicitly clears their history

---

#### 5. **Controller Layer** (`ProductViewController.java`)
REST API endpoints for client interaction.

**Base URL:** `/api/product-views`

## Frontend Integration Guide

### 1. Generate Guest Session ID
The frontend should generate a unique session ID for each guest user:

```typescript
// angular-service.ts
import { v4 as uuidv4 } from 'uuid';

@Injectable({
  providedIn: 'root'
})
export class ProductViewService {
  private guestSessionId: string;

  constructor(private http: HttpClient) {
    // Get from localStorage or generate new
    this.guestSessionId = this.getOrCreateSessionId();
  }

  private getOrCreateSessionId(): string {
    const key = 'guest_session_id';
    let sessionId = localStorage.getItem(key);
    
    if (!sessionId) {
      sessionId = uuidv4();
      localStorage.setItem(key, sessionId);
    }
    
    return sessionId;
  }

  logProductView(productId: number): Observable<any> {
    return this.http.post('/api/product-views/log', {
      guestSessionId: this.guestSessionId,
      productId: productId
    });
  }

  getRecentlyViewed(): Observable<any> {
    return this.http.get(`/api/product-views/recently-viewed`, {
      params: { guestSessionId: this.guestSessionId }
    });
  }

  getRecentlyViewedDetailed(): Observable<any> {
    return this.http.get(`/api/product-views/recently-viewed/detailed`, {
      params: { guestSessionId: this.guestSessionId }
    });
  }

  clearRecentlyViewed(): Observable<void> {
    return this.http.delete<void>(`/api/product-views/clear`, {
      params: { guestSessionId: this.guestSessionId }
    });
  }
}
```

### 2. Log View on Product Detail Page

```typescript
// product-detail.component.ts
export class ProductDetailComponent implements OnInit {
  productId: number;
  product: any;

  constructor(
    private route: ActivatedRoute,
    private productService: ProductService,
    private productViewService: ProductViewService
  ) {}

  ngOnInit(): void {
    this.route.params.subscribe(params => {
      this.productId = params['id'];
      this.loadProduct();
      this.logView();
    });
  }

  loadProduct(): void {
    this.productService.getProduct(this.productId).subscribe(product => {
      this.product = product;
    });
  }

  logView(): void {
    // Log the view silently (no need to wait for response)
    this.productViewService.logProductView(this.productId).subscribe({
      error: (err) => console.error('Failed to log view:', err)
    });
  }
}
```

### 3. Display Recently Viewed Strip

```typescript
// recently-viewed.component.ts
@Component({
  selector: 'app-recently-viewed',
  template: `
    <div class="recently-viewed-strip" *ngIf="products && products.length > 0">
      <h3>Recently Viewed</h3>
      <div class="products-scroll">
        <div class="product-item" *ngFor="let item of products">
          <img [src]="item.productImage" [alt]="item.productName">
          <h4>{{ item.productName }}</h4>
          <p class="price">${{ item.productPrice | currency }}</p>
          <button (click)="viewProduct(item.productId)">View</button>
        </div>
      </div>
      <button (click)="clearHistory" class="clear-btn">Clear History</button>
    </div>
  `
})
export class RecentlyViewedComponent implements OnInit {
  products: any[] = [];

  constructor(
    private productViewService: ProductViewService,
    private router: Router
  ) {}

  ngOnInit(): void {
    this.loadRecentlyViewed();
  }

  loadRecentlyViewed(): void {
    this.productViewService.getRecentlyViewed().subscribe(response => {
      this.products = response.products;
    });
  }

  viewProduct(productId: number): void {
    this.router.navigate(['/product', productId]);
  }

  clearHistory(): void {
    if (confirm('Clear recently viewed products?')) {
      this.productViewService.clearRecentlyViewed().subscribe(() => {
        this.products = [];
      });
    }
  }
}
```

## Key Features & Behavior

### Deduplication
- When a user views a product they've already seen:
  - The product is NOT duplicated in the list
  - Timestamp is updated to current time
  - Product moves to position 1 (front of list)
  - View count is incremented

### Max 10 Products
- Only the 10 most recent products are stored
- Automatic cleanup when exceeding limit
- Oldest products are automatically deleted

### Session-Based Tracking
- Each guest has a unique `guestSessionId` (UUID)
- Session persists across page navigation
- Session persists across browser refresh (stored in localStorage)
- Session ends when user clears browser data or generates new session

### Order
- Always ordered by most recent first (newest timestamp first)
- Used for horizontal scrolling strip on Homepage/PDP

## Database Schema

```sql
CREATE TABLE product_views (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    guest_session_id VARCHAR(255) NOT NULL,
    product_id BIGINT NOT NULL,
    viewed_at TIMESTAMP NOT NULL,
    view_count INT NOT NULL DEFAULT 1,
    FOREIGN KEY (product_id) REFERENCES products(id),
    INDEX idx_session_time (guest_session_id, viewed_at DESC)
);
```

## Performance Considerations

1. **Database Indexing:**
   - Index on `(guest_session_id, viewed_at DESC)` for fast retrieval
   - Included in ProductViewRepository queries

2. **Pagination:**
   - Service uses `PageRequest.of(0, 10)` for efficient query
   - Prevents loading unnecessary data

3. **Lazy Loading:**
   - ProductView→Product relationship uses `FetchType.EAGER`
   - Necessary for serving product data in single query

4. **Cleanup Strategy:**
   - Automatic deletion happens on each view log
   - Prevents table bloat over time
